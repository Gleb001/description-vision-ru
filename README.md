- # 🎥 Vision Video Hosting Platform

## О проекте

**Vision** — это современная платформа для хостинга видео, которую я разработал в рамках дипломного проекта. Она предоставляет полный набор инструментов для загрузки, хранения, воспроизведения и управления видеоконтентом — по сути, это полноценный аналог популярных видеохостингов, но с акцентом на кастомизацию и масштабируемость.

Архитектурно проект построен на микросервисном подходе, что позволило разделить систему на независимые компоненты. В основе лежат несколько сервисов: API Gateway на Node.js и Express.js, который выступает единой точкой входа и занимается маршрутизацией и проксированием запросов; сервис аутентификации, написанный на PHP, который отвечает за генерацию и верификацию JWT-токенов; пользовательский сервис на Node.js, управляющий профилями и подписками; а также S3-хранилище, тоже на Node.js с использованием Multer, которое обрабатывает загрузку и потоковую отдачу видео и изображений. Фронтенд-часть разбита на три независимых приложения на Next.js — это основной плеер, творческая студия для авторов и личный кабинет пользователя. Благодаря микросервисной структуре каждый компонент можно обновлять и масштабировать независимо, что делает систему гибкой и отказоустойчивой.

Что касается технологического разнообразия, здесь действительно смесь разных подходов: от PHP в одном микросервисе до современных фреймворков вроде Next.js и React на фронтенде. Всё это связано через REST API, а хранение данных обеспечивает MySQL, причём каждый сервис работает со своей собственной базой, что дополнительно повышает изоляцию.

Инфраструктурная часть тоже продумана до мелочей. На сервере развёрнут Docker, а управление контейнерами автоматизировано через Docker Compose. В качестве reverse proxy выступает Nginx, который принимает все внешние запросы, распределяет нагрузку между контейнерами и обеспечивает безопасное SSL-соединение. А чтобы не тратить время на ручной деплой, настроен полноценный CI/CD-пайплайн через GitHub Actions — теперь при каждом пуше в основную ветку автоматически собираются Docker-образы, отправляются в приватный реестр, а затем обновляются на продакшн-сервере.

В итоге получилась не просто учебная работа, а живая, готовая к эксплуатации система, которая объединяет в себе современный стек, гибкую архитектуру и продуманную инфраструктуру.

## 🏗 Архитектура

### Микросервисная структура

#### 1. **API Gateway (api.vision-ru.ru)**
- **Технологии**: Node.js + Express.js, TypeScript
- **Функции**:
  - Единая точка входа для всех клиентских запросов
  - Маршрутизация на основе паттернов (регулярные выражения)
  - Проверка JWT-токенов с автоматическим обновлением через refresh-токены
  - Проксирование запросов к соответствующим сервисам
  - Управление cookies и CORS
  - Поддержка chunked-передач для потоковой передачи видео

#### 2. **Auth Service (auth.vision-ru.ru)**
- **Технологии**: PHP 8+, MySQL
- **Функции**:
  - Генерация JWT-токенов (access + refresh)
  - Верификация токенов (HS256 алгоритм подписи)
  - Обновление токенов через refresh-механизм
  - Завершение сессий пользователей
  - Хранение refresh-токенов в базе данных

#### 3. **User Service (us.vision-ru.ru)**
- **Технологии**: Node.js + Express.js, TypeScript, MySQL
- **Функции**:
  - Регистрация и аутентификация пользователей
  - Управление профилями (аватар, био, отображаемое имя)
  - Система подписок (подписки/подписчики)
  - Получение детальной информации о пользователях
  - Валидация JWT-токенов через внешний сервис

#### 4. **S3 Storage Service (storage.vision-ru.ru)**
- **Технологии**: Node.js + Express.js, Multer, MySQL
- **Функции**:
  - Загрузка видео и изображений (multipart/form-data)
  - Потоковая передача с поддержкой HTTP Range запросов
  - Автоматическое определение MIME-типов
  - Кэширование статического контента (Cache-Control)
  - Учет просмотров (авторизованные + анонимные пользователи)
  - Управление уровнями доступа (public, limited, link)
  - Возрастные ограничения (0+, 6+, 12+, 16+, 18+)

#### 5. **Основной видеоплеер (vision-ru.ru)**
- **Назначение** — главная точка входа для зрителей. Обеспечивает просмотр контента, навигацию по каталогу и взаимодействие с сообществом.
- **Технологии**: Next.js + React, TypeScript, Redux Toolkit

#### 6. **Творческая студия пользователя для работы со своим контентом (studio.vision-ru.ru)**
- **Назначение** — инструментарий для авторов контента. Позволяет загружать видео, управлять библиотекой и отслеживать аналитику.
- **Технологии**: Next.js + React, TypeScript, Redux Toolkit

#### 7. **Аккаунт пользователя (id.vision-ru.ru)**
- **Назначение** — управление личными данными, безопасностью и социальными связями. Служит единой точкой входа для всех сервисов платформы.
- **Технологии**: Next.js + React, Typescript, Redux

### Архитектура клиента

// Основные слайсы

Особенности реализации
Гибридный рендеринг:

- SSR для динамических страниц (поиск, профиль)
- SSG для статических страниц (главная)
- ISR для постепенного обновления контента

Видеоплеер:
- Кастомный плеер на основе HTML5 Video API
- Интерактивный прогресс-бар с превью кадров
- Система горячих клавиш
- Автоматическое запоминание позиции просмотра
- Адаптивное качество (HLS/DASH)

Оптимизация:
- Lazy-loading компонентов и маршрутов
- Мемоизация селекторов (createSelector)
- Базовая стратегия кэширования
- Skeleton-анимация (useStyleShimmer)

Безопасность:
- httpOnly cookies для хранения JWT-токенов
- CSRF-защита
- Валидация форм (Formik + кастомная валидация)
- Автоматическое обновление токенов через интерцепторы

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| API Gateway | Node.js + Express.js | Маршрутизация, проксирование |
| Auth Service | PHP 8+ | JWT-генерация, верификация |
| User Service | Node.js + Express.js | CRUD профилей, подписки |
| Storage Service | Node.js + Express.js + Multer | Загрузка, хранение, стриминг |
| База данных | MySQL 8.0.42 | Хранение данных |
| Контейнеризация | Docker | Изоляция, воспроизводимость |
| Оркестрация | Docker Compose | Управление контейнерами |
| Reverse Proxy | Nginx 1.25.3 | Балансировка, SSL-терминация |

CI/CD Pipeline (GitHub Actions)
- Push в ветку main
- Автоматическая сборка Docker-образов
- Публикация в приватный registry (registry.vision-ru.ru)
- Автоматический деплой на production-сервер через SSH

Безопасность
- Firewall: UFW (блокировка всех портов, кроме 22, 80, 443)
- SSH: Только ключевая аутентификация, нестандартный порт (2222)
- TLS: Let's Encrypt с автоматическим обновлением, TLS 1.2/1.3
- Контейнеры: Seccomp-профили, read-only корневые FS, cgroups-ограничения
- База данных: Параметризованные запросы (защита от SQL-инъекций)

🔐 Безопасность
- JWT-токены: Хранение в httpOnly cookies (защита от XSS)
- Refresh-механизм: Автоматическое обновление access-токенов
- CSRF-защита: Проверка origin через CORS-заголовки
- SQL-инъекции: Только параметризованные запросы
- XSS: Экранирование пользовательского ввода
- CSP: Content-Security-Policy заголовки
- HTTPS: Все соединения через TLS 1.2/1.3
